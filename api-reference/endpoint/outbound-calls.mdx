# Outbound & Scheduled Calls API

## Make an Outbound Call

Initiate a single or batch outbound call.

```http
POST /outbound-call
```

### Request

<CodeGroup>
<CodeGroupItem title="Single Call">

```json
{
  "to_number": [
    {
      "phone_number": "+91 7874527239",
      "name": "Raju",
      "email": "",
      "company": "",
      "notes": ""
    }
  ],
  "from_number": "+918031272965"
}
```

</CodeGroupItem>

<CodeGroupItem title="Batch Call">

```json
{
  "to_number": [
    {
      "phone_number": "+91 78745 27239",
      "name": "Raju",
      "email": "",
      "company": "",
      "notes": ""
    },
    {
      "phone_number": "+9199097832",
      "name": "Shyam",
      "email": "",
      "company": "",
      "notes": ""
    }
  ],
  "from_number": "+918031272965",
  "name": "Raju999"
}
```

</CodeGroupItem>
</CodeGroup>

## Schedule a Call

Schedule a single or batch call for a future time.

```http
POST /schedule-call
```

### Request

<CodeGroup>
<CodeGroupItem title="Single Call">

```json
{
  "scheduled_time": "2025-10-30T12:40:00.587Z",
  "call_request": {
    "to_number": [
      {
        "phone_number": "+91 7874527239",
        "name": "Raju",
        "email": "",
        "company": "",
        "notes": ""
      }
    ],
    "from_number": "+918031272965"
  }
}
```

</CodeGroupItem>

## Best Practices

1. **Input Validation**
   - Always validate phone numbers before making requests
   - Ensure proper country codes are included
   - Validate email formats if provided

2. **Error Handling**
   - Check for `success` flag in responses
   - Handle partial success cases in batch operations
   - Implement retry logic for transient failures

3. **Rate Limiting**
   - Be aware of API rate limits
   - Implement exponential backoff for retries
   - Queue large batch operations

4. **Security**
   - Never expose API keys in client-side code
   - Use environment variables for sensitive data
   - Validate all responses before processing
